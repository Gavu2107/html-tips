10 Lessons from Your HTML Mentor

1. Understand, Don’t Just Copy: Before using a tag or property, learn *why* it exists. Don’t just paste code from the internet — experiment with it to understand how it works.
2. Write Semantic HTML: Use meaningful tags like <header>, <main>, <section>, and <footer>. This helps search engines and screen readers understand your page better.
3. Build for Everyone: Add alt text for images, label your forms, and make sure your colors have good contrast. Accessibility ensures your website can be used by *everyone*.
4. Keep It Clean: Use proper indentation and spacing. Clean code isn’t just pretty — it’s easier to debug, maintain, and share with others.
5. Validate and Learn: Use the W3C HTML Validator to check your work. It helps you catch small errors and understand the correct HTML structure.
6. Inspect and Explore: Right-click any webpage and choose “Inspect.” You’ll see how professionals build layouts — a great way to learn modern techniques.
7. Think Responsively: Add the viewport meta tag and use relative units (like %, em, rem). Test your design on mobile and tablet screens to ensure it looks great everywhere.
8. Build, Break, Repeat: Don’t fear mistakes! The best way to learn HTML is by experimenting — build small projects, break them, then fix them again.
9. Comment with Care: Use comments (<!-- like this -->) to describe your sections or tricky code. It helps you and others understand your project later.
10. Never Stop Being Curious: HTML is the foundation of the web. Explore new tags, follow tutorials, and try building real projects — every experiment makes you better.
